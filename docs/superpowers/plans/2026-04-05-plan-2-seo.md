# SEO Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Agregar hreflang, og:locale:alternate, canonical en servicios, JSON-LD Service schema, y crear la pagina /en/cita que faltaba.

**Architecture:** SEO.astro acepta nuevos props `lang` y `altUrl` para generar hreflang dinamicamente. La pagina [slug].astro pasa canonical y schema Service. Se crea /en/cita.astro como mirror de /cita.astro.

**Tech Stack:** Astro 5, Schema.org, Open Graph

### Task 2: Agregar canonical y JSON-LD Service en paginas de servicios

**Files:**
- Modify: `src/pages/servicios/[slug].astro`

- [ ] **Step 1: Actualizar el frontmatter de [slug].astro**

Reemplazar el bloque frontmatter completo (lineas 1-20) con:

```astro
---
import { getCollection } from 'astro:content';
import BaseLayout from '../../layouts/BaseLayout.astro';
import BookingButton from '../../components/BookingButton.astro';
import CalendarIcon from '../../components/icons/CalendarIcon.astro';

export async function getStaticPaths() {
  const services = await getCollection('services');
  return services.map((service) => ({
    params: { slug: service.slug },
    props: { service },
  }));
}

const { service } = Astro.props;
const { Content } = await service.render();

const calLink = service.data.calLink || import.meta.env.PUBLIC_CAL_LINK || 'daniel-torres-calvo';

const serviceSchema = {
  '@context': 'https://schema.org',
  '@type': 'Service',
  name: service.data.title,
  description: service.data.description,
  image: service.data.image,
  provider: {
    '@type': 'LocalBusiness',
    name: 'Blue Royale Beauty & Spa',
    url: 'https://www.blueroyalespa.com',
  },
};
---
```

- [ ] **Step 2: Actualizar la llamada a BaseLayout para pasar canonical y jsonLd**
